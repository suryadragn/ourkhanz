# SIMRS-Khanza Database Schema Reference

## Table Directory

### Core Registration Tables
- [pcare_pendaftaran](#pcare_pendaftaran) - Main registration table
- [booking_registrasi](#booking_registrasi) - Pre-registration bookings
- [reg_periksa](#reg_periksa) - Examination records

### Master Data Tables
- [jadwal](#jadwal) - Doctor schedules
- [dokter](#dokter) - Doctor information
- [poliklinik](#poliklinik) - Clinic information
- [pasien](#pasien) - Patient information

---

## pcare_pendaftaran

**Purpose:** Main registration table storing all patient registrations  
**Scope:** Hospital-wide  
**Cardinality:** 1 registration per patient per date  

### Key Fields

| Field | Type | Constraint | Description |
|-------|------|-----------|-------------|
| `no_rawat` | VARCHAR(17) | PRIMARY KEY | Registration number (Format: YYYY/MM/DD/NNNNN) |
| `tglDaftar` | DATE | NOT NULL, INDEX | Registration date |
| `no_rkm_medis` | VARCHAR(15) | FK, NOT NULL | Patient medical record number |
| `nm_pasien` | VARCHAR(40) | NOT NULL | Patient name (denormalized for speed) |
| `kdPoli` | VARCHAR(5) | FK, NOT NULL | Clinic code |
| `nmPoli` | VARCHAR(30) | NOT NULL | Clinic name (denormalized) |
| `noUrut` | VARCHAR(5) | NOT NULL, INDEX | Queue number (A1, B5, C12, etc.) |
| `status` | VARCHAR(50) | ENUM | 'Masuk', 'Pulang', 'Pindah', etc. |
| `sistolik` | INT | | Systolic blood pressure |
| `diastolik` | INT | | Diastolic blood pressure |
| `tinggi_badan` | INT | | Height in cm |
| `berat_badan` | DECIMAL(5,2) | | Weight in kg |
| `suhu_tubuh` | DECIMAL(5,2) | | Body temperature |
| `tekanan_nadi` | INT | | Heart rate/pulse |
| `pernafasan` | INT | | Respiratory rate |
| `gula_darah` | DECIMAL(5,2) | | Blood glucose level |

### Indices

```sql
PRIMARY KEY (no_rawat)
INDEX idx_tgldaftar (tglDaftar)
INDEX idx_norkmmedis_tgldaftar (no_rkm_medis, tglDaftar)
INDEX idx_noUrut (noUrut)
INDEX idx_kdpoli_tgldaftar (kdPoli, tglDaftar)
```

### Foreign Keys

```sql
FOREIGN KEY (no_rkm_medis) REFERENCES pasien(no_rkm_medis) ON DELETE CASCADE
FOREIGN KEY (kdPoli) REFERENCES poliklinik(kd_poli) ON DELETE CASCADE
```

### Queue Number Assignment

- **Format**: `[CLINIC_PREFIX][SEQUENCE]`
- **Example**: A1, A2, A3, B1, B2, etc.
- **Scope**: Per clinic per date
- **Reset**: Daily (counter resets at midnight)
- **Generated By**: Database trigger or stored procedure
- **Not in Application Code**: Handled at database layer

## booking_registrasi

**Purpose:** Temporary bookings before actual patient registration  
**Scope:** Pre-registration state  
**Cardinality:** 1 booking per patient per appointment date  

### Key Fields

| Field | Type | Constraint | Description |
|-------|------|-----------|-------------|
| `tanggal_booking` | DATE | NOT NULL | When booking was made |
| `jam_booking` | TIME | NOT NULL | Booking time |
| `no_rkm_medis` | VARCHAR(15) | PK, FK | Patient medical record number |
| `tanggal_periksa` | DATE | PK, NOT NULL | Appointment date |
| `kd_dokter` | VARCHAR(20) | FK, NOT NULL | Doctor code |
| `kd_poli` | VARCHAR(5) | FK, NOT NULL | Clinic code |
| `no_reg` | VARCHAR(8) | | Registration number (NULL until registered) |
| `kd_pj` | CHAR(3) | FK | Payment method (default 'A09') |
| `limit_reg` | INT | | Registration limit (default 0) |
| `waktu_kunjungan` | DATETIME | | Expected visit datetime |
| `status` | VARCHAR(20) | ENUM | 'Belum', 'Terdaftar', 'Batal', 'Dokter Berhalangan' |

### Indices

```sql
PRIMARY KEY (no_rkm_medis, tanggal_periksa)
INDEX idx_tanggalperiksa (tanggal_periksa)
INDEX idx_kddokter (kd_dokter)
INDEX idx_kdpoli (kd_poli)
INDEX idx_status (status)
```

### Foreign Keys

```sql
FOREIGN KEY (no_rkm_medis) REFERENCES pasien(no_rkm_medis) ON DELETE CASCADE
FOREIGN KEY (kd_dokter) REFERENCES dokter(kd_dokter) ON DELETE CASCADE
FOREIGN KEY (kd_poli) REFERENCES poliklinik(kd_poli) ON DELETE CASCADE
FOREIGN KEY (kd_pj) REFERENCES penjab(kd_pj) ON DELETE CASCADE
```

### Status Values

- **'Belum'** - Booking created, not yet registered
- **'Terdaftar'** - Patient has checked in and registered
- **'Batal'** - Booking cancelled (does not count in quota)
- **'Dokter Berhalangan'** - Doctor unavailable (does not count in quota)

### Quota Impact

- When booking is created: `jadwal.kuota` decremented by 1
- When status = 'Batal' or 'Dokter Berhalangan': **NOT counted** in remaining quota
- Multiple bookings with same (no_rkm_medis, tanggal_periksa) prevented by PRIMARY KEY

## reg_periksa

**Purpose:** Examination/consultation records  
**Scope:** Linked to pcare_pendaftaran registration  
**Cardinality:** 1+ per registration (one per visit/consultation)  

### Key Fields

| Field | Type | Constraint | Description |
|-------|------|-----------|-------------|
| `no_rawat` | VARCHAR(17) | FK, PK | Registration number |
| `kd_dokter` | VARCHAR(20) | FK | Doctor code |
| `jam_reg` | TIME | | Examination start time |
| `tgl_reg` | DATE | | Examination date |
| `stts` | VARCHAR(20) | ENUM | 'Belum', 'Sudah', 'Sudah Anamnesa', 'Batal' |

### Foreign Keys

```sql
FOREIGN KEY (no_rawat) REFERENCES pcare_pendaftaran(no_rawat) ON DELETE CASCADE
FOREIGN KEY (kd_dokter) REFERENCES dokter(kd_dokter) ON DELETE CASCADE
```

### Status Values

- **'Belum'** - Waiting to be examined
- **'Sudah'** - Examination completed
- **'Sudah Anamnesa'** - Anamnesis (medical history intake) completed
- **'Batal'** - Cancelled examination

## jadwal

**Purpose:** Doctor availability schedules  
**Scope:** Master schedule defining clinic capacity  
**Cardinality:** 1+ per doctor (one per working day per clinic)  

### Key Fields

| Field | Type | Constraint | Description |
|-------|------|-----------|-------------|
| `kd_dokter` | VARCHAR(20) | PK, FK | Doctor code |
| `hari_kerja` | VARCHAR(10) | PK | Working day (SENIN-AKHAD) |
| `jam_mulai` | TIME | PK | Schedule start time |
| `jam_selesai` | TIME | | Schedule end time |
| `kd_poli` | VARCHAR(5) | FK | Clinic code |
| `kuota` | INT | NOT NULL | Daily patient capacity |

### Indices

```sql
PRIMARY KEY (kd_dokter, hari_kerja, jam_mulai)
INDEX idx_harikerja (hari_kerja)
INDEX idx_kdpoli (kd_poli)
```

### Foreign Keys

```sql
FOREIGN KEY (kd_dokter) REFERENCES dokter(kd_dokter) ON DELETE CASCADE
FOREIGN KEY (kd_poli) REFERENCES poliklinik(kd_poli) ON DELETE CASCADE
```

### Days of Week (hari_kerja)

```
SENIN    - Monday
SELASA   - Tuesday
RABU     - Wednesday
KAMIS    - Thursday
JUMAT    - Friday
SABTU    - Saturday
AKHAD    - Sunday
```

### Quota Behavior

- **Initial Value**: Set by admin (typically 5-10)
- **Decremented**: When booking is confirmed
- **Cannot Go Below Zero**: Enforced by `WHERE kuota > 0` in UPDATE query
- **Restored**: Only on manual admin action (booking cancellation)

## Query Patterns

### Get Available Schedules with Remaining Quota

```sql
SELECT 
  j.kd_dokter,
  d.nm_dokter,
  j.jam_mulai,
  j.jam_selesai,
  j.kuota,
  COALESCE(b.jumlah_pendaftar, 0) AS booked_count,
  GREATEST(j.kuota - COALESCE(b.jumlah_pendaftar, 0), 0) AS kuota_tersisa
FROM jadwal j
LEFT JOIN dokter d ON d.kd_dokter = j.kd_dokter
LEFT JOIN (
  SELECT kd_dokter, COUNT(*) AS jumlah_pendaftar
  FROM booking_registrasi
  WHERE tanggal_periksa = :tanggal
    AND status NOT IN ('Batal', 'Dokter Berhalangan')
  GROUP BY kd_dokter
) b ON b.kd_dokter = j.kd_dokter
WHERE j.hari_kerja = :hari_kerja
  AND j.kuota > 0
ORDER BY j.jam_mulai;
```

### Get Recent Registrations with Queue Info

```sql
SELECT 
  p.no_rawat,
  p.noUrut,
  p.nm_pasien,
  r.jam_reg,
  r.kd_dokter,
  d.nm_dokter,
  r.stts AS status_periksa
FROM pcare_pendaftaran p
LEFT JOIN reg_periksa r ON r.no_rawat = p.no_rawat
LEFT JOIN dokter d ON d.kd_dokter = r.kd_dokter
WHERE p.tglDaftar = :tanggal
ORDER BY p.noUrut ASC;
```

### Atomic Booking Operation

```sql
-- Step 1: Check & decrement quota (MUST be in transaction)
UPDATE jadwal
SET kuota = kuota - 1
WHERE kd_dokter = ?
  AND hari_kerja = ?
  AND jam_mulai = ?
  AND kuota > 0;

-- Step 2: Check if update succeeded (rows affected > 0)
-- If yes: proceed to Step 3
-- If no: ROLLBACK (quota exhausted)

-- Step 3: Insert booking
INSERT INTO booking_registrasi 
  (tanggal_booking, jam_booking, no_rkm_medis, tanggal_periksa, 
   kd_dokter, kd_poli, status)
VALUES (CURDATE(), CURTIME(), ?, ?, ?, ?, 'Belum');

-- Step 4: COMMIT (all or nothing)
```

## Performance Recommendations

### Indices to Create

```sql
-- booking_registrasi
CREATE INDEX idx_br_tanggal_status 
  ON booking_registrasi(tanggal_periksa, status);

CREATE INDEX idx_br_kddokter 
  ON booking_registrasi(kd_dokter, tanggal_periksa);

-- pcare_pendaftaran  
CREATE INDEX idx_pp_tglDaftar_kdPoli 
  ON pcare_pendaftaran(tglDaftar, kdPoli);

CREATE INDEX idx_pp_noUrut 
  ON pcare_pendaftaran(noUrut, tglDaftar);

-- jadwal
CREATE INDEX idx_jd_harikerja_kdpoli 
  ON jadwal(hari_kerja, kd_poli);
```

### Partitioning Strategy

For large hospitals with years of data:

```sql
-- Partition pcare_pendaftaran by month
ALTER TABLE pcare_pendaftaran 
PARTITION BY RANGE (YEAR(tglDaftar) * 100 + MONTH(tglDaftar)) (
  PARTITION p202601 VALUES LESS THAN (202602),
  PARTITION p202602 VALUES LESS THAN (202603),
  -- ... etc
  PARTITION pmax VALUES LESS THAN MAXVALUE
);
```
